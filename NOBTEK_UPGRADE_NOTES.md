# Nobtek upgraded build

این نسخه برای محیط تست شما آماده شده و `db.sqlite3` و `.venv` طبق درخواست حذف نشده‌اند.

## تغییرات اصلی

- بازنویسی موتور زمان‌های آزاد با پشتیبانی واقعی از `buffer_minutes`.
- بررسی تداخل رزروها، ظرفیت خدمت، زمان استراحت، زمان‌های بسته و تداخل کارمند.
- ثبت رزرو عمومی داخل `transaction.atomic()` و قفل ردیف خدمت/کارمند برای کاهش ریسک رزرو همزمان در دیتابیس‌های production مثل PostgreSQL.
- امن‌سازی لغو رزرو با token امضاشده و محدودیت POST.
- تغییر پرداخت آزمایشی از GET به POST.
- افزودن لایه سرویس برای رزرو، پرداخت آزمایشی، اعلان‌ها و توکن لغو.
- افزودن پایه اتصال پیامک و پرداخت از طریق تنظیمات env.
- اعتبارسنجی موبایل ایران، تصویر، ساعت کاری، زمان بسته، قیمت، بیعانه، ظرفیت و محدودیت‌های پلن.
- اعمال محدودیت‌های پلن برای تعداد کارمند، تعداد خدمت، پرداخت آنلاین، بیعانه و ظرفیت رزرو ماهانه.
- افزودن مدیریت کارمندها در داشبورد.
- افزودن نمای تقویم ۳۰ روز آینده.
- افزودن گزارش ماهانه پایه.
- بهبود SEO برای صفحات پیگیری رزرو با `noindex,nofollow`.
- افزودن لاگینگ production-friendly.
- افزودن تست‌های رزرو، امنیت لغو، پرداخت آزمایشی، فرم موبایل و smoke test داشبورد.

## بررسی‌های انجام‌شده

```bash
python manage.py check
python manage.py makemigrations --check --dry-run
python manage.py test
DJANGO_DEBUG=False ... python manage.py check --deploy
```

همه موارد بالا در محیط بررسی پاس شده‌اند.

## نکته production

درگاه بانکی و پیامک واقعی بدون کلید API و انتخاب سرویس‌دهنده نمی‌توانند عملیاتی شوند. در این نسخه ساختار امن و آماده اتصال اضافه شده است:

- `bookings/payments.py`
- `bookings/notifications.py`
- تنظیمات `NOBTEK_PAYMENT_GATEWAY` و `NOBTEK_SMS_BACKEND`

برای انتشار واقعی، مقدار `DJANGO_SECRET_KEY` را حتماً با یک مقدار طولانی و تصادفی جایگزین کنید و SQLite را به PostgreSQL تغییر دهید.
